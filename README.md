## 氣泡排序法

原理:

氣泡排序法是一種簡單的排序演算法。它會重複地走訪待排序的資料列，依序比較相鄰的兩個元素，若順序錯誤則交換，直到整個資料列有序為止。

複雜度分析:

| 項目             | 複雜度 | 說明               |
|------------------|--------|--------------------|
| 最壞時間複雜度   | O(n²)  | 陣列完全反向排序時 |
| 平均時間複雜度   | O(n²)  | 隨機排序的情況     |
| 最佳時間複雜度   | O(n)   | 陣列已排序時       |
| 空間複雜度       | O(1)   | 只需常數大小的額外空間 |

## 選擇排序法

原理:

選擇排序法是一種簡單的排序演算法。它會在未排序的序列中找到最小（或最大）元素，存放到排序序列的起始位置，然後再從剩餘未排序元素中繼續尋找最小元素，然後放到已排序序列的末尾。以此類推，直到所有元素均排序完畢。

複雜度分析:

| 項目             | 複雜度 | 說明               |
|------------------|--------|--------------------|
| 最壞時間複雜度   | O(n²)  | 無論輸入資料如何   |
| 平均時間複雜度   | O(n²)  | 隨機排序的情況     |
| 最佳時間複雜度   | O(n²)  | 即使已排序也需完整比較 |
| 空間複雜度       | O(1)   | 只需常數大小的額外空間 |

模擬實作: https://catherine-liu0730.github.io/sort_report/selection_sort.html

## 插入排序法

原理:

插入排序法是一種基於比較的排序演算法。它透過構建一個已排序的序列，然後逐個將未排序的元素插入到已排序序列的適當位置，直到整個序列有序為止。

複雜度分析:

| 項目             | 複雜度 | 說明               |
|------------------|--------|--------------------|
| 最壞時間複雜度   | O(n²)  | 陣列完全反向排序時 |
| 平均時間複雜度   | O(n²)  | 隨機排序的情況     |
| 最佳時間複雜度   | O(n)   | 陣列已排序時       |
| 空間複雜度       | O(1)   | 只需常數大小的額外空間 |

模擬實作: https://catherine-liu0730.github.io/sort_report/insertion_sort.html

## 合併排序法

原理:

合併排序法是一種高效的排序演算法，採用分治法的思想。它將待排序的資料列逐次分割成較小的部分，對各部分進行排序，最後將各部分合併成一個有序的序列。

複雜度分析:

| 項目             | 複雜度        | 說明                           |
|------------------|---------------|-------------------------------|
| 最壞時間複雜度   | O(n log n)    | 任意情況下都是這個複雜度       |
| 平均時間複雜度   | O(n log n)    | 隨機排序的情況                 |
| 最佳時間複雜度   | O(n log n)    | 陣列已排序時仍為 O(n log n)    |
| 空間複雜度       | O(n)          | 需要領外的空間儲存臨時子陣列   |

模擬實作: https://catherine-liu0730.github.io/sort_report/merge_sort.html

## 快速排序法

原理:

快速排序法是一種高效的排序演算法，採用分治策略。它選擇一個樞紐元素，將陣列分為兩部分：小於樞紐的元素和大於樞紐的元素，然後遞歸地對兩部分排序。

複雜度分析:

| 項目             | 複雜度               | 說明                                      |
|---------------------|----------------------|------------------------------------------|
| 最壞時間複雜度   | O(n²)                | 樞紐選擇不當時（如總選擇最大/最小值）  |
| 平均時間複雜度   | O(n log n)           | 樞紐選擇較好的一般情況                  |
| 最佳時間複雜度   | O(n log n)           | 樞紐絕是分割點的中位數時                |
| 空間複雜度       | O(log n)             | 遞歸呼叫棧的深度                        |

模擬實作: https://catherine-liu0730.github.io/sort_report/quick_sort.html

## 綜合比較

### 時間複雜度比較

根據各演算法的複雜度分析，可得到以下結論：

1. **簡單排序演算法**：氣泡排序法、選擇排序法和插入排序法均屬於初級排序演算法，其平均時間複雜度均為 O(n²)，適用於小規模資料集的排序。其中，氣泡排序法在最佳情況下（陣列已排序）可達到 O(n) 的線性時間複雜度，具有一定的自適應性。

2. **高效排序演算法**：合併排序法和快速排序法均採用分治策略，平均時間複雜度為 O(n log n)，適用於大規模資料集的排序。然而，快速排序法在最壞情況下時間複雜度退化為 O(n²)，而合併排序法則保證在任何情況下都維持 O(n log n) 的時間複雜度，具有更高的穩定性。

### 空間複雜度比較

- **原地排序**：氣泡排序法、選擇排序法和插入排序法均為原地排序演算法，空間複雜度為 O(1)，不需佔用額外的一設記憶體。
- **額外空間需求**：合併排序法需要 O(n) 的額外空間用於臨時子陣列的儲存；快速排序法需要 O(log n) 的額外空間用於遞歸呼叫棧。

### 演算法穩定性分析

- **穩定排序**：氣泡排序法、插入排序法和合併排序法為穩定排序演算法，相同元素的相對位置在排序後不會改變。
- **不穩定排序**：選擇排序法和快速排序法為不穩定排序演算法，無法保證相同元素的相對順序。

### 應用場景建議

| 場景                   | 推薦演算法         | 原因                                           |
|------------------------|------------------|------------------------------------------------|
| 小規模資料（<50）     | 插入排序法或氣泡排序 | 實現簡單，開銷小，效率足夠                    |
| 中等規模資料（50-1000） | 快速排序法         | 平均效率最高，實踐中表現最佳                  |
| 大規模資料（>1000）   | 快速排序法或合併排序 | 快速排序平均效率高；合併排序保證性能穩定    |
| 需要穩定排序         | 合併排序法或插入排序 | 保證相同元素的相對位置不變                    |
| 記憶體有限           | 氣泡/選擇/插入排序 | 空間複雜度為 O(1)，佔用最少記憶體              |

### 結論

不同的排序演算法各具特色，選擇合適的演算法需要考慮資料規模、記憶體限制、穩定性需求等多方面因素。在實際應用中，快速排序法因其優秀的平均性能和廣泛的適用性而最為常用，而合併排序法則因其可預測的性能在對穩定性要求較高的場景中被優先選擇。
